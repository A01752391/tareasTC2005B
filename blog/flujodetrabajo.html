Flujo de trabajo de git y github

Regresar al inicio


El flujo de trabajo de Git y GitHub está basado en un método para estandarizar y mejorar la colaboración en proyectos que involucren en el trabajo en equipo.


Git

Concretamente, para el flujo de trabajo de git se utilizan branches, ya que estas permiten la colaboración y estandarización en proyectos.

  • Se utilizan las ramas para gestionar diferentes versiones del proyecto sin interferir en la rama principal y que, eventualmente, se puedan fusionar. Lo que da
    lugar para trabajar de manera aislada sin interferir con el trabajo de los demás.
  • Es de utilidad para sincronizar el código en el servidor remoto con el código en la computadora.
  • Facilita la colaboración al permitir que los integrantes trabajen y vean el código.


GitHub

En cuanto al flujo de trabajo en GitHub, igual se basa en el uso de branches y es adecuado para equipos grandes. Ya que, permite gestionar tareas, errores y nuevas características de forma más eficiente.

  • Es una herramienta para estandarizar cómo funciona el equipo y la forma en que utilizan GitHub.
  • Permite usar plantillas de flujo de trabajo para crear el flujo de trabajo personalizado.
  • Permite examinar la lista completa de plantillas de flujo de trabajo en el repositorio.


Comandos importantes

Comando Función
$ git status Muestra el estado actual del repositorio.
$ git add name_file.html Git comenzará a trackear el archivo y lo añadirá staging area.
$ git commit -m "Mensaje significativo." Crea un commit de los archivos en el staging area.
$ git push -u origin nombre_rama Actualiza los commits locales al repositorio remoto.
$ git switch nombre_rama Cambiar de rama.
$ git merge nombre_rama Fusionar ramas.
